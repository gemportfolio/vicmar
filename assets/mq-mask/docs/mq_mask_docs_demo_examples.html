mq-mask

A minimal CSS utility for masking (hiding) elements using precise screen widths. Compose ranges. Don’t memorize breakpoints.

How It Works

mq-mask provides only two utilities per width:

.mq-none-{width}-below /* hides at ≤ width */
.mq-none-{width}-above /* hides at ≥ width + 1 */

Combine both classes to hide elements between two screen widths.

Examples

Mobile-only element

<div class="mq-none-480-above">...</div>
Visible ≤ 480px

Desktop-only element

<div class="mq-none-768-below">...</div>
Visible ≥ 769px

Tablet-only element

<div class="mq-none-600-below mq-none-900-above">...</div>
Visible 601–900px

Hidden between 401px – 600px

<div class="mq-none-400-below mq-none-600-above">...</div>
Hidden between 401–600px

Hidden below 550px

<div class="mq-none-550-above">...</div>
Hidden below 550px

Hidden above 750px

<div class="mq-none-750-below">...</div>
Hidden above 750px

Supported Widths

mq-mask supports every pixel width between:

320px → 1200px

This allows precise control without relying on abstract breakpoint names like sm, md, or lg.
